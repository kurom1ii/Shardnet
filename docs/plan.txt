# Shardnet Plan

## Tổng quan
Shardnet = Giao thức giúp AI Agent tìm & kết nối tới Shard (đơn vị tri thức thực thi được).
Không blockchain, không P2P, không VPS nặng.

---

## Kiến trúc

```
AI Agent
  │
  ├─ 1. Embed query (local, CPU)
  │
  ├─ 2. Route đến domain phù hợp
  │     Tải manifest.json (2KB, cache local)
  │     cosine(query, centroid từng domain) → chọn domain
  │
  ├─ 3. Tải index.hnsw của domain đó
  │     URL tĩnh → GitHub Pages / R2
  │     cache local, ETag check cập nhật
  │
  ├─ 4. ANN search local (HNSW)
  │     index.hnsw → top-K ShardInfo
  │     ← { name, hash_id, tags, endpoint, public_key }
  │
  └─ 5. Kết nối & tương tác Shard
        shard_connect(endpoint) → HTTP / WebSocket
        shard_execute(hash_id)  → chạy code, load context
```

---

## Các thành phần

### Storage (chỉ là file tĩnh, không cần server)

| File | Nội dung | Kích thước |
|------|----------|------------|
| `manifest.json` | Danh sách domain + centroid + URL + version | ~2KB |
| `index.code.hnsw` | HNSW index cho domain "code" | ~3-10MB |
| `index.text.hnsw` | HNSW index cho domain "text" | ~3-10MB |
| ... | ... | ... |

Host trên GitHub Pages hoặc Cloudflare R2 — miễn phí, CDN toàn cầu, zero server.

### Client-side (shard-cli)

| Bước | Tốn gì | Lần đầu | Lần sau (cache) |
|------|--------|---------|-----------------|
| Embed query | CPU ~10ms | ✓ | ✓ |
| Tải manifest | Network ~2KB | ✓ | ETag ~304 |
| Route (cosine centroid) | CPU <0.01ms | ✓ | ✓ |
| Tải index | Network ~3-10MB | ✓ | ETag ~304 |
| ANN search | CPU <1ms | ✓ | ✓ |

### User tự host Shard (bảo mật)

```
Máy user → Cloudflare Tunnel → Internet
  - Không mở port, không lộ IP
  - TLS tự động, chống scan, chống DDoS
  - ShardInfo.endpoint = public URL cloudflare, không phải IP thật
  - Vector tấn công duy nhất: chính shard đó (lỗi logic, RCE)
```

---

## Bảo mật & Trust

- **SHA-512**: Xác minh nội dung toàn vẹn (không collision)
- **Ed25519 Signature**: Xác minh tác giả (public key = identity)
- **Trust model**: Local whitelist — mỗi agent tự quản lý key tin cậy
- **Key mới / lạ**: Cảnh báo, user confirm, không tự động trust
- **Không**: Web-of-trust toàn cầu, stake, blockchain
